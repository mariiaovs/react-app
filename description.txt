Основная проблема появления объекта undefined в ProductPage.tsx 
это установление в качестве начального значения пустой объект в файле product.ts 
const [product, setProduct] = useState<IProduct>(пустой объект);

Далее насколько вызовом метода fetchProduct продукт заменяется на объект полученный
по "https://fakestoreapi.com/products/1", но поскольку fetchProduct функция асинхронная
и Promise обрабатывается первый возврат return { product, loading, error }; успевает 
вывалиться с пустым объектом и порушить тем самым логику в ProductPage. 
Простых решения на вскидку 2:
1. Либо устанавливать какой-то начальный продукт в 
const [product, setProduct] = useState<IProduct>(продукт по умолчанию);
в файле product.ts 
2. Либо отыгрывать появление пустого продукта непосредственно в ProductPage, я поставил 
return (<div>Loading product ...</div>); 
но можно вставить какой-нибудь спиннер и т.п.

Но это на вскидку, может есть какое-то стандартное решение использования асинхронного 
метода в useEffect.
